## 1. 基本信息

- 名称：简易版 Offer Show 微信公众号后台系统
- 目的：提供一个可匿名提交 Offer、并按条件查询/查看详情的轻量平台
- 版本：1.0.0
- 创建日期：2026.04.26

## 2. Offer Show 实现

- 功能 1：找名企 -> 名企实习校招
  - 功能描述：用户可以通过公众号菜单或关键词，快速获取知名企业的实时招聘动态（包括实习生招聘和正式校招）。
  - 交互逻辑：
    - 用户点击底部菜单“找名企”中的子菜单“名企实习校招”。
    - 系统自动回复当前最新的企业招聘列表（图文或列表形式）。
    - 支持用户直接输入“企业名”进行特定搜索。
  - 展现内容：
    - 企业名称（如：腾讯、华为、美团）。
    - 职位类别（如：技术类、产品类、运营类）。
    - 工作地点。
    - 招聘链接或详情入口。
- 功能 2：查薪资
  - 功能描述：用户通过输入“公司名称”或“职位关键字”，查询该企业在市场上的具体薪资爆料。
  - 交互逻辑：
    - 用户在对话框直接发送：“查薪资 字节跳动”。
    - 系统匹配数据库中的薪资记录。
    - 若匹配到多条，则以分页列表形式返回。
  - 展现内容：
    - 公司名称与具体部门。
    - 岗位名称（如：Java 开发工程师）。
    - 薪资详情：包括月薪、发薪月数、年终奖、签字费等（例如：22k * 16 + 3w）。
    - 备注：档位信息（白菜、SP、SSP）及学历背景。

## 3. 需求范围

### 3.1 目标用户

- 在校大学生、职场人士、待业人员

### 3.2 功能需求

- 提交 Offer
  - `commit <公司> <城市> <岗位> <薪资>`
  - `group-commit <公司1> <城市1> <岗位1> <薪资1> [公司2 城市2 岗位2 薪资2 ...]`
- 查询 Offer（分页列表）
  - `query [--company 公司] [--city 城市] [--position 岗位] [--page 页码] [--sort-new] [--sort-salary]`
  - company/city/position 支持“包含匹配”（模糊查询）
  - 当查询结果为空时，返回“没有找到对应的 Offer。”
- 查看单条详情
  - `detail <id>`：显示与 edit 成功后相同的详情信息（不展示创建时间/用户）
- 编辑与删除
  - `edit <id> [--company <公司>] [--city <城市>] [--position <岗位>] [--salary <薪资>]`
  - `edit <id> <公司> <城市> <岗位> <薪资>`
  - `delete <id>` / `delete --all`
  - 权限：只能操作自己提交的 Offer
- 辅助功能
  - `help` / `help <command>`
  - `ping`
  - `stats`（我的统计）
  - `my`（只看我提交的）
  - `latest`（最新列表）

### 3.3 非功能需求（已实现/约束）

- 高可用
  - 后端异常时尽量不影响微信服务器的基本回调（返回可解析 XML）
- 权限与反滥用
  - 高频请求检测：在固定时间窗口内超过阈值则标记为爬虫用户并拒绝后续查询能力
- 输入校验
  - 薪资必须是整数；当薪资参数不合法时，提示“参数格式不对，请检查是不是把薪资写成了整数。”

## 4. 功能详细说明（命令式交互）

### 4.1 输入输出形式

- 输入：用户在公众号聊天窗口输入一行文本命令
- 输出：后端返回多行文本，微信客户端展示为普通消息

### 4.2 关键约束

- 参数使用空格分隔；公司/城市/岗位不应包含空格
- `query` 用于列表查询；单条详情请使用 `detail`
